壹、緣起

貳、專案概述

一、專案名稱
平安商會互助會系統建置案

二、專案目標
本專案旨在為「平安商會互助會」開發一個全面的數位化營運平台，分為使用者前台與管理者後台，以提升會員服務品質與內部管理效率。

1.  **使用者前台系統 (Member-Facing Frontend):**
    *   **優化使用者體驗：** 打造一個直觀、易於操作的介面，讓會員在不同裝置（電腦、手機）上都能獲得一致且流暢的體驗。
    *   **核心功能實作：** 提供會員線上參與互助會（標會）、查詢個人帳務明細、管理個人資料等功能，實現操作便利性。
    *   **資訊透明化：** 確保會員能隨時清楚查閱錢包餘額、資金流向與互助會進度，提升信任感。

2.  **管理者後台系統 (Administrative Backend):**
    *   **提升管理效率：** 建立一個集中化的管理平台，使管理人員能高效地處理日常營運事務。
    *   **強化管理功能：** 提供對會員資料、互助會期、網站內容（如圖片）及金流的全面管理與監控能力。
    *   **確保資料安全與準確性：** 建立安全的資料管理機制，進行財務稽核，確保系統營運的順暢與數據準確。

三、 專案範圍
本專案旨在建置「平安商會互助會系統」的前端使用者介面及相應的管理者操作介面，以實現專案目標中所述的各項功能。

**1 功能性範圍 (Functional Scope):**
*   **使用者前台 (Member Frontend):**
    *   會員註冊與登入功能：包含一般帳號密碼及串接LINE ID第三方登入。
    *   商會大廳：瀏覽、參與現有互助會活動。
    *   會員錢包：帳戶餘額查詢、轉入註冊點數、加值、提領餘額、金流明細、死會明細。
    *   會員資訊：變更密碼、參與標案、組織、KYC。
*   **管理者後台 (Administrative Backend Interface):**
    *   內容管理：管理前台顯示圖片、公告等。
    *   標會管理：標會狀態查詢。
    *   會員管理：會員資料及錢包查詢。
    *   權限管理：KYC審核。
    *   帳務管理：儲值及提領管理、帳務紀錄查詢。

**2 非功能性範圍 (Non-Functional Scope):**
*   **現代化的技術：** 我們採用穩定、主流的網頁技術來打造您的平台，確保系統未來容易擴充與維護。
*   **高度的安全性：** 我們遵循網頁安全最佳實踐，例如將資料加密傳輸，透過嚴謹的加密機制確保資料傳輸的機密性，以保護貴會與會員的資料安全。
*   **支援多元裝置：** 網站會自動適應電腦、平板、手機等不同尺寸的螢幕，確保在任何裝置上都有良好的視覺和操作體驗。

**3. 排除範圍 (Out-of-Scope):**
*   **原生行動應用程式 (Native Mobile Applications)：** 不包含 iOS 或 Android 原生應用程式的開發。
*   **複雜報表分析與商業智慧 (BI) 系統：** 僅提供基本帳務數據的展示，不包含進階的數據分析與預測功能。
*   **支付閘道或金流串接的核心實作：** 錢包功能僅限於數據展示與簡單操作介面，不包含與第三方支付系統的直接串接實作。
*   **詳細的資安滲透測試與認證：** 僅提供基礎安全規劃，不包含深度的資安顧問服務或第三方資安認證。

參、需求建議
一、 系統架構及環境說明
（一） 整體系統運作環境架構規劃建議。
為滿足貴會「維持舊有操作習慣」與「升級系統」的雙重需求，我們將採用穩定且具彈性的現代化架構。您可以將整個系統視為一個具備清晰分工的整合體：

*   **會員網站 (前端應用):**
    這是會員每天接觸到的地方。我們將沿用現有的設計風格，讓會員感覺親切、操作順手，無需重新學習。

*   **管理者後台 (管理應用):**
    這是為您的管理團隊打造的全新指揮中心。我們將採用與會員網站一致的技術來建置，使其操作流暢、容易上手，方便您統一管理所有業務。
    獨立的管理者登入機制(跟一般使用者分開管理)、二次驗證的機制(後台有出入金相關功能，降低資料外洩所造成的風險)

*   **系統核心 (後端服務與資料儲存):**
    這是系統的後端邏輯與資料庫，負責所有運算與資料儲存。我們將採用穩定、安全的 **C#** 與 **PostgreSQL** 技術，確保整個平台運作高效、資料安全無虞。

（二） 整體系統安全環境規劃與建議。
我們極度重視系統的安全性，並規劃了以下措施：

*   **資料加密傳輸：**
    會員的所有資料在網路上傳輸時，都會經過國際標準的加密程序，透過國際標準的加密技術，有效防止未經授權的第三方在資料傳輸過程中竊取敏感資訊。

*   **權限分級管理：**
    系統會嚴格區分不同人員的權限，確保會員只能存取自己的資料，而不同層級的管理者也只能操作其職責範圍內的項目，滴水不漏地保護個資與系統安全。

*   **軟體版本管理：**
    我們會持續關注所有構成系統的軟體版本，並定期進行更新與安全修補，以防範已知的安全漏洞。

二、 系統開發及建置說明
（一） 系統規劃及設計方法（含系統安全、效能、操作、控制、整合等功能特性）。
我們採用模組化開發方法，這是一種將系統拆解成獨立功能模組的策略。系統的每一個功能（例如登入、錢包、會員資料）都是一個獨立且標準化的積木。
此策略帶來以下優勢：
1.  **開發效率高**：我們可以專注於打造每一個功能積木，然後快速地將它們組合起來。
2.  **維護成本低**：如果未來某個小功能需要調整，我們只需針對那一塊積木進行修改，而不會影響到整個系統。
3.  **擴充彈性大**：未來若想新增功能，只需開發新的功能積木並加入系統即可，讓系統能與時俱進。

（二） 開發建置解決方案描述，與所遵循之標準。
為了打造一個高品質且長久穩定的系統，我們選用以下業界廣受信賴的技術與標準：

*   **使用者介面技術 (Vue.js):**
    我們選擇 Vue.js 來打造使用者看得到、摸得著的操作介面。它以打造流暢、反應快速的互動效果而聞名，能帶給會員不輸一線網站的現代化體驗。

*   **系統核心技術 (C# / .NET):**
    系統的大腦與核心邏輯，我們選用由微軟(Microsoft)開發的 C# 語言。它在安全性、執行效能方面表現卓越，特別適合處理需要高度穩定與安全的商業應用。

*   **資料庫技術 (PostgreSQL):**
    所有寶貴的資料都將存放在 PostgreSQL 資料庫中。這是一款以穩定和資料完整性著稱的頂級開源資料庫，確保您的數據被妥善且安全地保管。

*   **開發流程標準 (版本控制與統一風格):**
    我們的開發過程遵循嚴謹的標準。
    1.  **版本控制(Git):** 我們會為程式碼建立一個「開發日誌」，所有變更都被詳細記錄。此機制確保開發過程中的所有修改均詳實記錄，提供完整的版本歷史追溯能力，提升開發作業的可靠性與安全性。
    2.  **統一風格:** 團隊成員會遵循一致的開發規範，這有助於確保最終產出的系統結構清晰、品質一致，為未來的維護奠定良好基礎。

（三） 系統功能與管理維護機制之描述。
新系統將包含以下主要功能，為會員提供完整的服務閉環：
- **首頁 (Home)**：作為網站的門面，提供最新的商會消息與活動資訊。
- **登入頁 (Login)**：會員進入個人化服務的入口，支援便利的 LINE 帳號快速登入。
- **互助會大廳 (Hall)**：清晰展示所有可參與的互助會活動，方便會員輕鬆加入。
- **會員中心 (Account)**：會員的個人專區，可在此管理個人資料與查詢參與記錄。
- **我的錢包 (Wallet)**：會員的個人線上金庫，清楚記錄每一筆資金流向，安全又透明。
